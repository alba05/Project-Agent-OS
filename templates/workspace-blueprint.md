# Workspace Blueprint

Use this structure for a new project workspace.

```text
project-name/
  README.md
  knowledge-base/
    current-state.md
    open-loops.md
    decision-log.md
    sop-index.md
  sop/
    startup-sop.md
  thread-summaries/
    thread-001.md
  scripts/
  assets/
  deliverables/
```

## Notes

1. Keep the knowledge base small and active.
2. Use thread summaries for session-to-session continuity.
3. Put repeatable procedures in `sop/`.
4. Avoid mixing raw experiments into the knowledge base.
